Java Application Suite Failures

Project: Circles vs. Squares — Falling Through Hole Logic

Vulnerability: Misalignment between collision detection and gravity state updates.

System Failure: The player character would frequently ignore floor bounds and clip through platform edges when approaching at specific velocity vectors, causing a failure to trigger ground-state resets[cite: 3].

Resolution: Refactored the vertical collision resolution to prioritize positional snap-backs to the platform top boundary immediately upon overlap detection, ensuring strict floor-contact logic[cite: 3].

Project: Circles vs. Squares — Wall Climbing Logic

Vulnerability: Lack of boundary-check triggers for climbing inputs.

System Failure: Climbing inputs triggered erratic jittering or failed to initiate because the engine lacked a specific state to distinguish between standard platform collisions and vertical wall surfaces[cite: 1].

Resolution: Implemented a wall-check verification that validates proximity to vertical edges; once triggered, the engine forces a vertical movement shift to simulate vertical climbing mechanics[cite: 1].

Project: Circles vs. Squares — Ending Separation

Vulnerability: Improper serialization of game-state terminal nodes.

System Failure: During the transition to the game's finale, the engine failed to properly clear global variable buffers, leading to "ending separation" where the game would erroneously trigger both "Good" and "Bad" ending states simultaneously[cite: 1].

Resolution: Decoupled the ending selection logic by introducing a definitive terminal-state check that locks the game into a specific branch based on the total death count calculation performed during the session[cite: 1].

Computer Vision Integration Errors

Project: Drone Vision — Window Overlap & Stack

Vulnerability: Dynamic output window placement caused GUI collisions with mirrored capture regions.

System Failure: By default, OpenCV's window manager would stack the detection feed on top of the mirrored drone window, obstructing the source area and causing recursive capture errors[cite: 6].

Resolution: Implemented cv2.moveWindow to force the detection output to a hardcoded coordinate offset, physically separating the detection visualization from the capture region[cite: 6].

Project: Drone Vision — Detection Latency

Vulnerability: Uncalibrated loop frequency during high-resolution screen capture.

System Failure: Improper handling of video source retrieval in the mss grab loop caused significant latency in real-time object tracking[cite: 6].

Resolution: Shifted to a direct NumPy-based frame conversion for the screen buffer, drastically reducing the overhead between capture and the YOLO inference pass[cite: 6].
